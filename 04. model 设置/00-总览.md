# 04. model 设置

本目录用于放当前产品阶段已经足够稳定、可以开始被工程实现引用的数据对象模型草案。

当前包含三部分：

1. `训练计划模型`
2. `通用引用模型`
3. `训练笔记模型`
4. `训练实例模型`
5. `athlete_lift_profile`

补充说明：

- `训练计划模型` 已从单一大文件重构为一个目录，位于 [01-训练计划模型](/Users/heruoxin/dev/PRD/电子教练/04.%20model%20设置/01-%E8%AE%AD%E7%BB%83%E8%AE%A1%E5%88%92%E6%A8%A1%E5%9E%8B)。
- 该目录当前按对象层级拆分为：
  - 中周期
  - 小周期
  - 训练日
  - 训练项目
  - 训练组
  - 训练项目类型表
- 这样做是为了避免把“周期层级”“训练项目实例”“训练项目类型元数据”“单组处方”混成一个巨大的嵌套对象。

## 使用原则

- 这里的模型优先表达“持久化对象”。
- `撤销 / 重做 / 脏状态 / 当前选中 tab` 这类纯 UI 会话态，不放进这里。
- `训练执行结果` 不写进 `训练计划模型`，而是继续写进 `training_record`。
- `OS` 不属于训练计划本体；它是训练记录层的对象。
- 项目当前不存在“模板实体”；训练计划主要通过手动新建、复制既有计划或 AI 自动安排产生。

## `uid` 约定

- 所有正式对象都使用随机十六进制字符串作为稳定 `uid`。
- 不使用 `plan-001`、`note-001` 这种带业务语义的编号当主键。
- 训练笔记自身也使用同一套 `uid` 规则，便于后续 AI 教练模型直接引用。

## 引用模型约定

- 所有“可被别的对象引用”的内容，统一通过 `object_ref` 表达。
- `object_ref` 不只是一个机器可读指针，也要携带一份用户可见的卡片快照。
- 这份卡片快照至少包含：
  - 标题
  - 副标题
  - 可多行显示的正文片段
  - 页脚或跳转文案

这样做的目的有两点：

1. 后续 `AI教练页` 可以直接把引用渲染成卡片，而不必再临时拼展示文案。
2. 训练笔记、AI 对话、训练记录之间可以共享同一套引用结构。
