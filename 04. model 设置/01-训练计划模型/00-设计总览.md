# 训练计划模型重构总览

本目录用于彻底替换旧的单文件 `训练计划模型`。

这一轮的目标不是在旧架构上继续补丁，而是把训练计划域重新拆成一组职责单一、可独立读写、可扩展的正式对象。

## 当前范围

本轮只覆盖：

1. 中周期 `mesocycle`
2. 小周期 `microcycle`
3. 训练日 `plan_day`
4. 训练项目 `plan_exercise_item`
5. 训练组 `plan_set`
6. 训练项目类型表 `exercise_type_catalog`

本轮暂不单独设计：

- `macrocycle`
- 训练执行结果
- AI 对话模型

补充说明：

- 未来若要补 `macrocycle`，它只需要成为一个新的更高层对象，持有有序 `mesocycle_uids` 即可。
- 这也是为什么本轮刻意不把 `mesocycle` 再嵌进某个更大的 root object 中。

## 设计原则

### 1. 采用分层规范化对象，而不是一个巨大的嵌套计划树

- `mesocycle` 只负责中周期级信息与有序 `microcycle_uids`
- `microcycle` 只负责小周期级信息与有序 `day_uids`
- `plan_day` 只负责训练日级信息与有序 `exercise_item_uids`
- `plan_exercise_item` 只负责训练项目实例与有序 `set_uids`
- `plan_set` 只负责单组处方

这样做的好处：

- 编辑某一层时，不需要重写整棵计划树
- AI、笔记、训练记录都更容易稳定引用具体对象
- 后续导入导出、迁移、局部缓存都更清晰

### 2. 训练计划源与训练执行结果严格分离

- 本目录只表达“训练计划源”。
- `baseline_plan / working_plan / actual / set.state / training_record.state` 仍属于训练记录层。
- 训练开始后冻结出来的执行基线，不写回这里。

### 3. 训练项目类型表与训练项目实例分离

- `exercise_type_catalog` 负责记录：
  - 分类
  - 多语言名称
  - 默认记录档案
  - 强度参考主项
- `plan_exercise_item` 负责记录：
  - 某一天里具体用了哪个训练项目
  - 当前实例应用了哪份记录档案
  - 当前实例是否有字段可见性或强度映射覆盖

这能避免把“深蹲这个类型是什么”与“W3 D2A 这次深蹲怎么安排”混成一层。

### 4. `D` 标签按 exact string 处理

- `D1A`
- `D2A`
- `D2B`

这类标签在模型中直接按字符串存储，不再拆分出隐藏的族或索引结构。

### 5. `C / W` 标签显式持久化

- `mesocycle` 保留 `cycle_index / cycle_label`
- `microcycle` 保留 `week_index / week_label`
- 这样可以避免 UI 每次临时根据排序号拼接 `C1 / W3`
- 未来若存在复制、插入、重排或导入场景，也能把“用户看到的编号”与底层排序语义分开维护

### 6. `OS` 不属于训练计划源

- `OS` 是训练记录层的开放训练标签。
- 它不会出现在 `mesocycle / microcycle / plan_day` 这套可编辑训练计划对象里。
- 但训练日对象仍可通过 `creation_mode = converted_from_os` 记录“它曾经来自一次开放训练沉淀”。

### 7. 单组处方要支持“直接输入项 + 推导显示项”

当前 UI 已明确：

- 三大项默认围绕 `load / intensity / RPE` 工作
- 同一组优先只保留一个直接输入项
- 另外字段可由算法自动推导

因此 `plan_set` 不能只存一个平铺 `weight` 字段，而需要显式表达：

- 哪些字段是当前处方的一部分
- 哪个字段是本组的主要处方基准
- 哪些值是 authored
- 哪些值是 derived

### 8. 不把训练项目类型表做成 UI 模板库

- 项目中不存在“模板实体”。
- `exercise_type_catalog` 不是给用户直接浏览和实例化的模板页。
- 它是工程层的类型元数据表，用于驱动默认字段、名称映射和记录需求。

## 共同约定

### `uid`

- 所有正式对象都使用随机十六进制字符串。
- 类型表同时额外保留稳定的语义 `key`，便于工程引用与多语言映射。

### 有序关系

- 每一层父对象都保存一份有序 child uid 列表。
- 子对象也显式保存 parent uid。
- 这样既能做稳定装配，也能做高效局部读写。

### 范围语法

处方型数字统一沿用以下语法：

- `[5]`
- `[8, 10]`
- `[10, null]`

适用字段包括但不限于：

- `load.value`
- `intensity.percent_1rm`
- `rep.value`
- `rpe.value`
- `duration.value`
- `distance.value`

### 多语言名称

- 多语言名称使用 locale key 组织，例如：
  - `zh-CN`
  - `en`
- 后续若增加更多语言，只需要继续添加新的 locale 节点，而不需要改 schema 结构。
