# 首页.Tab02.计划页.计划详情与编辑页

## 页面定位

`计划详情与编辑页` 不应被理解为一个孤立的二级页，而应被理解为 `计划页` 这套复杂页面的“未开始 / 可编辑”状态。

它与 `训练中工作台`、`训练完成页` 共用同一套页面骨架，只是当前状态不同。

这一版的重点调整有三点：

- 组内不再支持拖拽排序，组序默认稳定向下。
- `推测` 不再作为单独栏目存在。
- 三大项的单组展示改为围绕 `load / intensity / RPE` 组织。

## 页面目标

- 让用户在一个稳定壳层中查看和编辑某个训练日。
- 让 `W3`、`D1A / D2A / D2B` 这类训练日标签既能切换，也能保持强上下文。
- 让动作与组次都以统一的表格型动作卡展示。
- 让编辑、保存、撤销、重做都发生在同一页面内，而不是频繁跳转。
- 保留动作卡拖拽排序，但弱化其视觉存在感，不鼓励高频改序。
- 让三大项默认以 `load / intensity / RPE` 三列表达计划强度。
- 让用户在开始训练前就能看到与当前训练日绑定的 `AI 分析 / 练前提醒`。

## 共同页面骨架

三个状态共用以下壳层：

1. 顶部导航栏
2. 周期与训练日切换栏
3. 纵向排列的动作卡列表
4. 页面底部的 `AI 分析 / 练前提醒`
5. 页面底部的 `+ 添加动作`

补充说明：

- 若首页壳层当前存在另一条进行中的训练，则本页顶部仍显示 [首页.全局训练高亮条.md](/Users/heruoxin/dev/PRD/电子教练/03.%20设计概览/首页.全局训练高亮条.md)。
- 用户点击该高亮条后，应直接回到对应训练的 `训练中工作台`。

### 顶部导航栏

基础形态如下：

```text
[返回] W3 D2A 卧推日     [撤销][重做] [问AI]     [保存]
```

规则说明：

- `返回` 回到 `计划页` 的来源视图与原位置。
- `撤销 / 重做` 作用于当前页面会话内尚未持久化的修改。
- 顶部 AI 按钮的文案状态统一遵循 [首页.Tab02.计划页.问AI按钮状态.md](/Users/heruoxin/dev/PRD/电子教练/03.%20设计概览/首页.Tab02.计划页.next/首页.Tab02.计划页.问AI按钮状态.md)。
- `保存` 只在页面进入脏状态后出现。
- 页面未修改时，`保存` 位置保持为空，不显示一个灰掉的假按钮。
- 当前页只保留会话级撤销与重做；保存后不保留长期版本历史。

### 周期与训练日切换栏

基础形态如下：

```text
[W3 ☰] D1A [D2A] D3A [+ 新增训练日]
```

规则说明：

- `W3 ☰` 可展开周期切换与新增周期。
- 训练日标签采用 tab 样式切换，直接显示计划中的 exact string，例如 `D1A`、`D2A`、`D2B`。
- `+ 新增训练日` 固定放在该行末尾，不放入更多菜单。
- 当前页面如果存在未保存修改，则不允许直接切换周期或训练日，必须先弹窗处理。
- 新增训练日时，系统可按当前计划习惯自动猜测建议名称，但最终保存的就是用户看到的那个字符串本身。
- 例如当前周已有 `D1A / D2A / D3A`，新增时可优先建议 `D4A`；若用户改成别的字符串，也应直接按该字符串保存。

### 动作卡列表

- 每个动作都是一张独立卡片。
- 卡片内部统一使用表格型布局。
- 每张卡片底部的 `+ 添加` 只负责在该动作末尾新增一组。
- 页面底部的 `+ 添加动作` 只负责把新动作追加到页面末尾。
- 动作卡支持拖拽排序，但拖拽手柄应弱化显示，不鼓励把它做成页面主操作。
- 同一动作卡内部的组序不再支持拖拽排序。

### 当前范围与边界

- 支持对动作进行拖拽排序。
- 不支持对动作内的组序进行拖拽排序。
- 支持在页面底部新增动作。
- 不存在所谓的 `休息块`。
- 组间休息只在训练中状态自然发生并记录，不作为计划页中的独立动作项出现。

## 页面规则

### 1. 页面默认就是统一编辑壳层

- 不再区分“普通用户模式”和“高阶用户模式”。
- 也不再区分“标准视图 / 表格视图”两套 UI。
- 首版统一采用当前这套表格型动作卡布局。

### 2. `保存`、`撤销`、`重做`

- 页面任意字段被修改后，页面进入脏状态。
- 脏状态下：
  - 显示 `保存`
  - `撤销 / 重做` 可用
- 无修改时：
  - 不显示 `保存`
  - `撤销 / 重做` 为空或弱化态
- 用户点击 `保存` 后，只持久化当前最终结果，不额外保留长期版本历史。

### 3. 顶部 `问AI`

- 顶部固定放置 AI 按钮。
- 按钮文案与未读逻辑统一遵循 [首页.Tab02.计划页.问AI按钮状态.md](/Users/heruoxin/dev/PRD/电子教练/03.%20设计概览/首页.Tab02.计划页.next/首页.Tab02.计划页.问AI按钮状态.md)。
- 点击后进入 `AI教练页`，并至少携带以下上下文：
  - 当前周期与训练日
  - 当前动作卡与组次内容
  - 当前 `AI 分析 / 练前提醒`
  - 与本次训练可能相关的用户笔记摘要

### 4. 周期与训练日切换

- 点击 `W3 ☰` 可展开：
  - 切换到其他周期
  - 新增周期
- 点击训练日标签切换同周期内不同训练日。
- 点击 `+ 新增训练日` 时：
  - 先给出一个按当前计划习惯推断的建议标签
  - 用户仍可直接改成任意新字符串
  - 系统不对 `D2A / D2B` 这类标签做隐藏归并
- 只要页面存在未保存修改：
  - 不允许直接切换周期
  - 不允许直接切换训练日
  - 不允许直接返回
- 此时统一弹窗提示：
  - `保存并切换`
  - `放弃修改`
  - `继续编辑`

### 5. 动作卡拖拽保留，但只保留动作级

- 长按动作卡头部的拖拽手柄，可对整张动作卡重新排序。
- 拖拽手柄默认应弱化显示，例如：
  - 使用低对比度颜色
  - 只放在卡头一角
  - 不给夸张的吸引注意视觉
- 同一动作卡内部的组序固定从上到下，不支持拖拽改序。
- 点击 `+ 添加` 时：
  - 默认在当前动作末尾新增一组
  - 新组自动继承上一组的字段结构与可见列
  - 但不强制继承上一组的所有数值
- 若当前动作还没有任何组，则用该动作的默认空行初始化。

### 6. 单组展示围绕 `load / intensity / RPE`

- 三大项和主项变式默认展示：
  - `load`
  - `intensity`
  - `次数`
  - `RPE`
  - `备注`
- 其中：
  - `load` 表示用户能直观看懂的负荷值，例如 `100kg`、`220lb`
  - `intensity` 表示当前组对应的 `e1RM` 百分比
  - `RPE` 表示目标主观强度
- `推测` 不再作为单独列存在。
- `%` 与 `RPE` 也不再作为 `load` 的特殊单位列去占据表格结构；它们分别归入 `intensity` 与 `RPE` 两列。

### 7. `load / intensity / RPE` 的联动规则

- 首版默认同一组优先只保留一个“直接输入项”。
- 当用户直接填写其中任意一个字段时：
  - 该字段显示为实心字
  - 另外两个字段根据当前用户参考数据自动推导
  - 推导结果使用弱化字或范围值展示
- 若用户随后又编辑另一个字段：
  - 最新被编辑的字段升级为新的“直接输入项”
  - 原本的直接输入项退回为推导显示
- 典型示例：
  - 用户填 `load = 100kg`
  - 系统推导 `intensity = 85%`
  - 系统推导 `RPE = 8.5~9.5`
- 若当前缺少足够参考数据：
  - 允许只显示用户填写项
  - 其余字段留空或显示轻量提示

### 8. 动作级设置对话框

- 动作中的 `[编辑]` 按钮点击后，弹出动作级对话框。
- 对话框修改的是该动作卡显示的栏目配置，而不是某一组的值。
- 首版可配置栏目至少包括：
  - `load`
  - `intensity`
  - `次数`
  - `RPE`
  - `备注`
  - `时间`
  - `距离`
- 对三大项和主项变式：
  - 默认启用 `load / intensity / RPE`
  - 默认保留“填一推二”的联动逻辑
- 对辅助项和有氧项：
  - 默认只展示必要列
  - 可按动作元数据决定是否显示 `intensity` 或 `RPE`

### 9. `load` 单元格本身承载单位

- 表格中不再保留独立的 `重量单位` 列。
- `load` 单元格本身同时承载：
  - 数值
  - 单位
- 点击 `load` 单元格后，弹出轻量编辑层。
- 首版至少支持：
  - `kg`
  - `lb`
  - `自重`
  - `辅助`
  - `弹力带`
- 对不同动作，允许用动作元数据控制可选项。

### 10. 新增能力的边界

- `+ 添加`：在当前动作末尾新增一组，并沿用当前动作的列配置。
- `+ 添加动作`：在页面最底部追加一个新动作。
- 新动作创建后，默认落在列表最末尾。
- 不支持在计划态通过拖拽重新排列组次；若用户想改顺序，应直接编辑已有组或重建组序。

### 11. `AI 分析 / 练前提醒`

- 页面底部固定放置 `AI 分析 / 练前提醒` 区域。
- 这块内容用于展示当前训练日的 AI 练前提醒，而不是训练后的总结。
- 提醒内容至少应覆盖：
  - 本次训练的执行重点
  - 当前计划中值得注意的风险点
  - 从用户笔记中索引到的、可能与本次训练相关的内容

刷新规则：

- 用户对当前训练日做出已保存修改后，`AI 分析 / 练前提醒` 需异步刷新。
- 页面存在未保存修改时，该区域显示轻量状态提示：
  - `保存后刷新练前提醒`
- AI 刷新中时，可显示：
  - `AI 正在更新本次训练提醒...`

开始训练前提醒：

- 无论用户是从 `计划页` FAB 还是从当前训练日进入训练中工作台，开始训练前都应再弹出一次练前提醒。
- 该提醒使用轻量弹层或底部抽屉展示最新摘要。
- 用户确认开始训练时，应同时冻结当前训练日的 `baseline_plan`。
- 用户至少可以选择：
  - `开始训练`
  - `进入 AI 对话`
  - `关闭`
- 这次提醒应优先复用当前页底部已生成的 `AI 分析 / 练前提醒`，避免重复生成。
- 若用户从这里进入 `AI 对话`，则该条练前提醒应作为该训练话题中的第一条特殊 AI 消息展示，并附带 `返回训练` 按钮。

### 12. 后台动作元数据表

- 本页不依赖图文动作库，但依赖一份后台动作元数据表。
- 这份数据库至少要解决三件事：
  - 某个动作默认显示哪些栏目
  - 某个动作支持哪些 `load` 录入方式
  - 某个动作是否支持 `intensity / RPE` 联动

首版建议把动作大致分为四类：

- `主项`
- `主项变式`
- `辅助项`
- `有氧运动`

每条动作元数据至少包含：

- `canonical_name`
- `aliases`
- `exercise_category`
- `default_columns`
- `supported_load_kinds`
- `supports_intensity`
- `supports_rpe`
- `reference_lift_key`
- `default_row_seed`

补充说明：

- 用户新建动作时，先按动作名匹配数据库。
- 命中后，自动生成该动作卡的默认列。
- 未命中时，回退到通用默认配置，并允许用户在动作设置里手动调整。

### 13. 表格列规则

不同动作类型的列允许轻量变化，但整体视觉节奏保持一致。

#### 三大项 / 主项变式

- `组次 | load | intensity | 次数 | RPE | 备注`

#### 常见辅助力量动作

- `组次 | load | 次数 | 备注`
- 若该动作明确需要强度跟踪，也可额外启用 `RPE`

#### 自重 / 稳定性 / 有氧动作

- `组次 | load | 次数/时间/距离 | 备注`

补充说明：

- `intensity` 首版主要服务三大项及其直接相关变式。
- 对不适合做 `e1RM` 百分比映射的动作，不强行显示 `intensity`。

## 与另外两个状态页的关系

- 切换到进行中的训练日 -> 进入 `训练中工作台`
- 切换到已结束训练日（`completed / terminated`） -> 进入 `训练完成页`
- 当前页是这套复杂页面的“可编辑计划态”

## 当前结论

`计划详情与编辑页` 当前应至少确认以下几点：

1. 它与另外两个页面共享同一套页面骨架。
2. 不再区分普通与高阶两种 UI 模式。
3. 页面脏状态下才显示 `保存`。
4. 未保存时不能切换周期、训练日或直接返回。
5. 只保留动作卡拖拽排序，不再支持组内拖拽排序。
6. 拖拽手柄需要弱化显示，不鼓励把排序当成高频主操作。
7. 三大项默认展示 `load / intensity / RPE`，不再保留单独的 `推测` 列。
8. `load` 单元格本身承载数值与单位，不再拆出单独 `单位` 列。
9. 同一组在 `load / intensity / RPE` 中优先只保留一个“直接输入项”，其余字段自动联动推导。
10. 顶部固定保留 AI 按钮，并统一遵循共用的 `问AI` 状态文档。
11. 页面底部固定有 `AI 分析 / 练前提醒`，且保存后异步刷新。
12. 开始训练前应再次向用户展示一次练前提醒。
13. 不存在休息块。
