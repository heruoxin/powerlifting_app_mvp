# 首页.Tab01.信息页

## 页面定位

信息页是用户进入 App 后默认看到的第一个主页面。

它不是泛首页，也不是训练记录列表的重复版本，而是一个围绕“当前训练决策”组织的 Widget 仪表板。

信息页要回答的核心问题是：

1. 我的 AI token 还剩多少，按当前速度还能用多久。
2. 我今天或下一次要练什么。
3. 我最近的状态和趋势如何。
4. 当前周期、最近一次训练和 AI 教练判断合在一起，最值得我注意的是什么。
5. 有哪些信息值得我现在立刻输入或处理。

## 页面目标

- 在首屏最上方明确展示用户问候、token 余额和预计可用时长。
- 用最短路径展示与当前训练最相关的信息。
- 用 Widget 机制承载不同用户对信息密度的偏好差异。
- 给未来 AI 自动生成的新型 Widget 预留稳定扩展位。
- 让用户可以在不进入深层页面的情况下完成少量高频输入。
- 让部分高价值卡片在正常浏览态下就具备较高视觉完成度，便于用户直接截图分享。

## 设计原则

### 1. 首屏第一张卡片固定承担问候与 token 转化职责

- 信息页最上方固定为系统级问候与 token 状态卡。
- 该卡片展示 `Hi, 某某某`、token 余额、近期消耗速度和预计可用时长。
- 该卡片固定置顶，不参与普通 Widget 排序。
- 该卡片可收起次级信息，但不应从首页完全移除。
- 该卡片右上角固定提供 `设置` 按钮，作为当前产品的全局设置入口。

### 2. 训练类内容仍是主要信息主体

- 除首屏固定卡片外，其余内容仍应优先服务训练前判断、训练中执行准备和训练后快速理解。
- 账户、点数以外的普通 Widget 排序，应以训练相关性为优先依据。

### 3. Widget 是信息页唯一主组织单位

- 信息页不再额外分频道、二级 tab 或宫格入口。
- 除首屏系统卡外，所有内容都以 Widget 卡片形式进入主滚动区。
- 页面本身只保留一个简洁头部、一个系统级首屏卡和一个 Widget 列表。

### 4. 首版采用单列纵向 Widget 流

- 为了保证排序、显隐、扩展和数据可读性，首版使用单列纵向排列。
- 不采用复杂瀑布流或自由拼贴布局。
- 小信息块可以作为单个 Widget 内的双列内容出现，但不作为独立拖拽单元拆分。

### 5. 允许个性化，但保留推荐默认顺序

- 用户可以排序和隐藏普通 Widget。
- 首次进入时使用推荐默认布局，帮助用户快速理解产品。
- 用户改动后，本地保存，不因升级或 AI 新增内容被强制重置。

### 6. 分享友好型卡片要兼顾社交展示效果

- 训练容量图、热力图、计划摘要、趋势图等卡片，不只是“能看懂”，还要达到“适合截图分享”的完成度。
- 分享友好型卡片要控制按钮密度，避免截图时充满操作控件。
- 视觉上应传达出自律、稳定、认真训练的气质，但不能伪造或夸大真实训练数据。

### 7. 玻璃风格服务于层次感，不干扰图表

- Widget 使用半透明玻璃卡片承载。
- 文本类 Widget 可以更通透。
- 图表类 Widget 需降低背景花纹和反射效果，保证线条、刻度和数值可辨认。

## 页面结构

信息页从上到下由四层组成：

1. 顶部标题区
2. 系统级问候与 token 状态卡
3. Widget 主列表
4. 编辑态下的 Widget 管理浮层

## 页面区块说明

### 顶部标题区

- 标题固定为 `信息页`。
- 右上角提供 `编辑` 入口。
- 副标题展示当前日期和训练周期定位信息，例如 `Base Week 3 Day 2`。
- 当没有周期信息时，副标题退化为日期和简短引导语。

### 系统级问候与 token 状态卡

- 固定显示在顶部标题区下方。
- 默认展示用户称呼、token 余额、近期消耗速度和预计可用时长。
- 右上角固定提供 `设置` 按钮，进入全局设置页。
- 提供 `购买` 和 `明细` 入口。
- 属于系统级卡片，不参与普通 Widget 排序。
- 允许折叠次级信息，但不建议支持彻底隐藏。

### Widget 主列表

- 采用单列纵向滚动。
- Widget 与 Widget 之间保持较明显的留白，避免玻璃卡片相互粘连。
- 每张 Widget 卡片都应可独立加载、独立失败和独立隐藏。

### 编辑态入口

- 点击 `编辑` 进入编辑态。
- 编辑态下页面不再展示 Widget 内部完整内容，只展示名称、排序手柄和显隐开关。
- 系统级问候与 token 状态卡单独展示为 `固定` 项，不进入拖拽排序区。
- 编辑态底部出现 `可添加 Widget` 区域和 `恢复默认布局` 操作。

## Widget 机制设计

### Widget 的基本属性

每个 Widget 至少具备以下属性：

- `id`
- `type`
- `title`
- `display_order`
- `is_visible`
- `data_state`
- `local_config`

说明：

- `type` 用于决定 Widget 的渲染方式和数据来源。
- `display_order` 用于拖拽排序。
- `is_visible` 控制是否在浏览态显示。
- `data_state` 至少区分 `ready / empty / loading / error`。
- `local_config` 用于存储时间范围、单位、是否显示预测线等个体设置。

### Widget 的用户操作

- `拖动排序`：长按后拖拽调整位置。
- `显示/隐藏`：通过开关控制是否出现在浏览态。
- `新增`：从 Widget 库中添加新的 Widget。
- `删除`：首版不提供真正删除，只提供隐藏，避免误删配置。
- `恢复默认`：恢复推荐顺序和推荐显隐状态。

补充规则：

- 系统级问候与 token 状态卡不参与上述普通 Widget 操作。
- 用户自定义能力作用于普通 Widget 列表，不作用于系统级首屏卡。

### Widget 的保存规则

- 用户对 Widget 布局的修改保存到本地。
- 排序和显隐是信息页级别配置，不应影响训练记录、笔记或 AI 页的结构。
- 后续若支持数据导出导入，Widget 配置应作为用户本地设置一并迁移。

### Widget 的跳转规则

- 每个 Widget 最多提供 `1 个主跳转 + 1 个次操作`。
- 主跳转用于进入更深层页面，例如训练记录详情、AI 教练对话、点数明细。
- 次操作用于当前卡片内的小动作，例如 `保存体重`、`切换时间范围`。
- 对标记为“可分享”的 Widget，`分享` 不进入单独页面，而是直接对当前页面执行长截图并唤起系统分享面板。
- 不允许一张 Widget 放入过多按钮，避免首页变成操作面板。

## 首批推荐默认 Widget 顺序

系统级固定首位：

1. `问候与 Token 状态卡`

普通 Widget 建议默认展示顺序如下：

1. `今日训练`
2. `周期与教练摘要`
3. `体重记录`
4. `三大项 PR / e1RM 趋势`
5. `训练容量热力图`

建议默认隐藏但可手动开启：

1. `快速备注`
2. `疼痛/异常记录`
3. `最近 AI 对话入口`

这样安排的原因：

- 顶部优先展示“今天练什么”和“AI 现在最关心什么”。
- 原先拆分的 `周期进度`、`最近训练总结`、`AI 教练焦点` 在首版中合并为一个 `周期与教练摘要`，减少语义重复。
- 但在普通 Widget 列表之前，系统级首屏卡固定展示 token 状态，以承担付费转化职责。
- 输入类首版仅保留更基础的 `体重记录`，减少首页复杂度。
- 趋势类和历史类 Widget 放在中下部，满足非训练时段回顾。
- 点数信息集中在系统级首屏卡内，不再作为默认独立普通 Widget 出现。

## 主要 Widget 种类设想

以下为信息页首批最重要的 Widget 类型池。

### 1. 综合摘要类 Widget

适合承载：

- `周期与教练摘要`
- `本周训练提醒`
- `计划变更提示`

特点：

- 以“当前周期定位 + 最近一次训练总结 + 当前 AI 焦点”三段合并内容为主体。
- 支持 `展开` 查看更完整内容。
- 默认提供 `分享` 次操作。

价值：

- 能把原本分散在周期进度、训练总结和 AI 提醒中的信息前置到首页。
- 减少多个语义相近 Widget 并排出现造成的割裂感。

### 2. 任务入口类 Widget

适合承载：

- `今日训练`
- `下一次训练`
- `待确认计划调整`

特点：

- 强调标题、目标内容、预估时长、当前阶段。
- 主按钮明确，例如 `开始训练`、`查看计划`、`接受调整`。
- 可显示“距离上次同类训练多久”的辅助信息。

价值：

- 直接服务训练闭环入口。
- 应作为信息页中最靠前的常驻 Widget 类型。

### 3. 横向容量图类 Widget

适合承载：

- `本周计划完成率`
- `恢复资源预算`（未来可选）

特点：

- 以横向进度条表达“已用 / 剩余 / 目标”关系。
- 进度条上可叠加分段色块，表达安全、提醒、临界状态。
- 适合呈现单一比值，不适合复杂趋势。

价值：

- 非常适合一眼判断“还有多少可用空间”。
- 对计划完成度都很直观。

### 4. 折线图类 Widget

适合承载：

- `三大项 PR 实际 / 推测趋势`
- `e1RM 变化`
- `体重趋势`
- `训练强度趋势`

特点：

- 一张卡片内最多展示 `1 到 3` 条线。
- 默认时间范围建议为 `最近 12 周`，可切换 `4 周 / 12 周 / 24 周`。
- 若同时展示“实际”和“推测”，需在视觉上明确区分实线与虚线。

价值：

- 用于表达长期变化趋势，而不是单次训练细节。
- 适合用户在训练间隙做长期回顾。

### 5. GitHub 风格绿点热力图类 Widget

适合承载：

- `训练容量热力图`
- `动作出勤图`
- `恢复打卡图`（未来可选）

特点：

- 首版明确采用 GitHub 风格的每日训练图。
- 以周为列、日为点展示近期密度。
- 颜色从浅到深表达当日训练容量，不宜使用过多色相。
- 首版不在该 Widget 内切换频次模式，先把“容量热力图”做清楚。

价值：

- 特别适合展示“练得够不够均匀”“最近是否断训”“某一周期是否过密”。
- 比纯列表更容易看见长期习惯模式。

### 6. 输入类 Widget

适合承载：

- `体重记录`
- `疼痛/异常记录`
- `训练前主观状态`

特点：

- 用户可以直接在 Widget 内输入并保存。
- 输入项数量必须克制，避免首页变成表单页。
- 保存后应出现即时反馈，例如轻微高亮、更新时间、成功提示。

价值：

- 让用户在首页就能完成最短输入链路。
- 为 AI 判断提供及时的当日状态数据。

### 7. 快捷操作类 Widget

适合承载：

- `快速备注`
- `问 AI 一个问题`
- `开始自由训练`

特点：

- 以一个简洁输入框或一个高优先级按钮为核心。
- 不展示大量历史，仅强调“立刻开始一个动作”。

价值：

- 适合承接高频且轻量的即时操作。
- 但优先级低于训练任务和状态判断类 Widget。

## 首批推荐 Widget 实例

### Widget 1：今日训练

- 类型：`任务入口类`
- 默认：显示
- 主要信息：
  - 今日训练标题与 `W1D2` 等训练日标签
  - 本次训练的精简计划块摘要
  - 预估时长
  - 与上次同类训练的间隔
- 主操作：`开始训练`
- 次操作：`查看计划`

### Widget 7：训练偏好

- 类型：`输入类 / 设置类`
- 默认：显示
- 主要信息：
  - 推荐每周几练
  - 默认重量单位 `kg / lb`
  - 对当前计划节奏的轻量说明
- 主操作：`编辑`
  - 以内联 sheet 或小弹层修改，不跳走
- 次操作：`查看计划页`

### Widget 2：周期与教练摘要

- 类型：`综合摘要类`
- 默认：显示
- 主要信息：
  - 当前周期所处位置
  - 最近一次训练总结的摘要
  - AI 对下一次训练的重点提醒
- 主操作：`展开`
- 次操作：`分享`
  - 直接对当前页面执行长截图并唤起系统分享面板

### Widget 3：体重记录

- 类型：`输入类`
- 默认：显示
- 主要信息：
  - 当前最近一次体重
  - 输入框
  - 保存按钮
- 主操作：`保存`
- 次操作：点击标题进入体重趋势详情

### Widget 4：三大项 PR / e1RM 趋势

- 类型：`折线图类`
- 默认：显示
- 主要信息：
  - 深蹲、卧推、硬拉的实际 PR 或推测 PR
  - 最近 `12 周` 趋势线
- 主操作：`查看详情`
- 次操作：切换 `实际 / 推测 / 同时显示`

### Widget 5：训练容量热力图

- 类型：`GitHub 风格绿点热力图类`
- 默认：显示
- 主要信息：
  - 最近 `12 周` 训练容量密度
  - GitHub 风格每日训练图
  - 色深对应当日训练容量
- 主操作：`查看详情`
- 次操作：`分享`
  - 直接对当前页面执行长截图并唤起系统分享面板

### Widget 6：快速备注

- 类型：`快捷操作类`
- 默认：隐藏
- 主要信息：
  - 一行输入框
  - 最近保存时间
- 主操作：`保存备注`
- 次操作：`查看全部笔记`

## Widget 的数据状态规则

### ready

- 数据完整，正常展示内容。

### empty

- 数据尚不足以构成 Widget。
- 不直接消失，而是展示引导态，告诉用户如何让它“变得有内容”。

### loading

- 采用骨架屏或简化占位，不闪烁。
- 各 Widget 独立加载，避免整页阻塞。

### error

- 仅该 Widget 显示轻量错误态。
- 提供 `重试` 或 `查看原因`，不影响其他 Widget。

## 当前结论

信息页首批应由一个系统级首屏卡，加上以下几类普通 Widget 建立能力基础：

- 系统级问候与 Token 状态卡

- 综合摘要类
- 任务入口类
- 折线图类
- GitHub 风格绿点热力图类
- 输入类
- 快捷操作类

其中最核心、最应该优先设计和实现的是：

1. `系统级问候与 Token 状态卡`
2. `今日训练`
3. `周期与教练摘要`
4. `体重记录`
5. `三大项 PR / e1RM 趋势`
6. `训练容量热力图`

这几类 Widget 已经足以构成一个真正服务力量举训练闭环的信息页，而不是泛化的健身首页。
