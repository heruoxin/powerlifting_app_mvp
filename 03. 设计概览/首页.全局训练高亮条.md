# 首页.全局训练高亮条

## 页面定位

`全局训练高亮条` 是首页壳层中的统一训练状态组件。

只要存在 `training_record.state = in_progress` 的训练，它就固定显示在首页壳层顶部，包括：

- `Tab01.信息页`
- `Tab02.计划页`
- `Tab03.训练笔记页`
- `Tab04.AI教练页`
- `计划详情与编辑页`
- `训练中工作台`
- 其他由首页壳层承载的相关展开页

它不只是“返回训练入口”，而是当前训练的统一状态条、计时条和定位入口。

## 设计目标

- 让用户在任意页面都能立刻知道训练是否仍在进行。
- 把“当前状态 + 当前段计时 + 总计时”收拢到一个全局组件里。
- 让 `训练中工作台` 不再重复显示一套页面内计时条或组间休息占位行。
- 让用户在离开工作台后，仍能一键回到当前训练。
- 让用户在工作台内，也能一键定位到当前正在进行的训练项目。

## 页面规则

### 1. 出现条件

- 只要存在进行中的训练，高亮条就显示。
- 它在 `训练中工作台` 内也不隐藏。
- 训练结束后，高亮条消失。
- 若用户从系统通知或灵动岛直接进入 `训练中工作台`，该条仍显示在壳层顶部。

### 2. 双行信息结构

- 第一行使用更大字号，显示“当前状态名”与“当前段计时”。
- 第二行使用较小字号，显示“计划上下文信息”与“总训练计时”。

第一行的状态名至少包括：

- 动作名，例如 `深蹲`
- `组间休息`
- `准备开始`

第二行的计划上下文至少包括：

- 来自既定计划时：
  - `W3 D2 Hypertrophy`
  - 或更完整的 `W3 D2 Hypertrophy / 卧推日`
- 来自开放训练时：
  - `W3 OS`

### 3. 计时规则

- 第一行右侧计时显示“当前段计时”：
  - 若当前有进行中的组，则显示该组已持续时间
  - 若当前处于组间休息，则显示本次休息已持续时间
  - 若训练刚启动但还未开始任何组，则显示 `00:00:00`
- 第二行右侧计时显示“本次训练总计时”：
  - 自训练开始起累计
  - 不因页面切换而重置

### 4. 点击行为

- 在非 `训练中工作台` 页面点击：
  - 直接进入当前训练对应的 `训练中工作台`
- 在 `训练中工作台` 页面点击：
  - 若当前存在进行中的训练组，则滚动并定位到该组
  - 若当前没有进行中的训练组，则滚动到页面中的第一个动作卡

大屏补充规则：

- 若右侧展开栏当前不是 `训练中工作台`，点击后在右栏打开当前训练工作台
- 若右栏已经是 `训练中工作台`，点击后只执行定位滚动，不重复打开

### 5. 与工作台页面的关系

- `训练中工作台` 中不再单独显示：
  - 页面级当前状态条
  - 页面级总计时条
  - 列表中的组间休息占位行
- 页面内只保留“当前进行中的训练组”高亮：
  - 行首箭头指针
  - 独特背景色
  - 独特字体颜色
- 当高亮条第一行显示 `组间休息` 时，页面列表本身不再插入一条假的休息行。

### 6. 视觉要求

- 高亮条应明显强于普通玻璃卡片，具备更高对比度与更明确的强调色。
- 两行文字都必须保持单行可快速扫读，不鼓励在条内堆叠按钮。
- 整条高亮条本身就是可点击区域，首版不强制单独放置 `返回` 或 `定位` 按钮。
- 第一行字号、字重、颜色优先级高于第二行。

## 当前结论

`全局训练高亮条` 当前应至少满足以下几点：

1. 它在训练进行中全局常驻，包括 `训练中工作台` 本身。
2. 它统一承载当前状态、当前段计时与总计时。
3. 它在非工作台页面承担“返回训练”入口，在工作台页面承担“定位当前训练项目”入口。
4. 它取代工作台内原本重复的休息行与页面级计时显示。
