# 首页

## 页面定位

首页是 App 的根级导航容器，不是单独的业务信息流页面。

它承担三件事：

- 承载全局底部 `4 tab` 导航。
- 承载训练中的全局返回入口。
- 作为 `Tab01.信息页`、`Tab02.计划页`、`Tab03.训练笔记页`、`Tab04.AI教练页` 的共同外壳。

首页的设计重点不是“做一个热闹的首页”，而是让用户在任意时刻都能快速回到当前最重要的训练任务。

## 信息架构

首页固定采用底部 `4 tab` 结构：

1. `Tab01.信息页`
2. `Tab02.计划页`
3. `Tab03.训练笔记页`
4. `Tab04.AI教练页`

补充说明：

- `Tab02` 的底部导航显示文案与文档文件名统一使用 `计划页`。

四个 tab 的职责边界如下：

- `信息页`：展示与当前训练决策最相关的聚合信息，以 Widget 形式组织。
- `计划页`：查看未来若干次训练计划、已完成训练记录，并进入训练详情与训练中工作台。
- `训练笔记页`：沉淀用户笔记与教练笔记，并承载 AI 记忆文件的可见入口。
- `AI教练页`：承载对话、追问、计划讨论、上下文引用与附件协作。

## 全局设计原则

### 1. 首页是“训练工作壳”，不是泛内容首页

- 首页的第一优先级是帮助用户进入训练、理解训练、返回训练。
- 不引入资讯流、社区流或与训练闭环无关的内容。
- 所有 tab 都应服务 `计划 -> 执行 -> 复盘 -> 长期观察 -> 下一轮调整` 的主流程。

### 2. 底部导航长期稳定

- 底部 `4 tab` 的位置固定，不允许用户自定义 tab 数量和顺序。
- 用户可自定义的是 `Tab01.信息页` 内部 Widget 的排序和显示，不是首页主导航结构。
- 保持主导航恒定，降低高频使用中的认知切换成本。

### 3. 视觉风格采用 Apple Liquid Glass 方向

首页壳层采用偏 `liquid glass` 的设计语言，但要明确以可读性优先：

- 顶部栏、底部 tab bar、悬浮按钮、浮层使用半透明玻璃材质。
- 背景使用柔和渐变和大面积模糊色块，营造层次感，不使用纯平底色。
- 卡片边缘采用圆角、高光边和轻微内发光，体现液态玻璃感。
- 动效以轻柔吸附、轻微放大、模糊变化为主，不使用夸张弹跳。
- 图表与数字信息区域降低背景干扰，必要时使用更实的半透明底，避免“好看但看不清”。

### 4. 训练中的全局状态条必须常驻

当存在“进行中的训练”时：

- 首页任何 tab 顶部都显示固定高亮条。
- 高亮条本身承担“当前训练状态 + 计时 + 快速定位”三重作用，而不只是一个返回按钮。
- 在非 `训练中工作台` 页面点击高亮条，直接进入当前训练的 `训练中工作台`。
- 在 `训练中工作台` 页面点击高亮条，滚动到当前进行中的训练项目；若当前没有进行中的项目，则滚动到第一个动作卡。
- 该入口优先级高于页面自己的次级提示条。
- 与移动端系统级持续通知、灵动岛入口形成一致的返回逻辑。
- 高亮条的详细结构与状态规则，统一见 [首页.全局训练高亮条.md](/Users/heruoxin/dev/PRD/电子教练/03.%20设计概览/首页.全局训练高亮条.md)。

### 5. 大屏设备使用分栏而不是手机式底部壳层

- 在 `Pad` 等大屏设备上，首页主导航不再使用底部 tab bar。
- 改为左侧固定的类 `navigation rails` 布局。
- 左侧 rails 仍固定承载四个一级入口：
  - `信息`
  - `计划`
  - `笔记`
  - `教练`

大屏下各 tab 的适配原则如下：

- `信息页`
  - Widget 默认由“一行一个”改为“一行两个”网格排布。
  - 点击 Widget 展开详情时，不优先整页跳转。
  - 首选在当前壳层中央或偏右位置弹出全局悬浮 `pop up window` 承载展开页。
- `计划页`
  - 采用双栏到三栏的工作布局。
  - 左侧主栏显示时间轴 / 月视图，以及时间轴顶部的 `W 指示器 / D 过滤器` 等控制区域。
  - 点击训练日后，`计划详情与编辑页`、`训练中工作台`、`训练完成页` 优先显示在最右侧展开栏，而不是覆盖整个页面。
- `训练笔记页`
  - 采用双栏布局。
  - 左栏显示笔记列表与顶部 tab。
  - 右栏显示当前打开的单条笔记详情、编辑态或 AI 教练笔记文件。
- `AI教练页`
  - 原本手机上的 `navigation drawer` 在大屏下默认展开。
  - 页面直接形成双栏：
    - 左栏为话题列表
    - 右栏为当前对话内容

### 6. 首页壳层不提供独立设置按钮

- 首页壳层本身不存在固定的右上角 `设置` 按钮。
- 全局设置入口由 `Tab01.信息页` 顶部固定的系统级问候与 token 状态卡承载。
- 用户点击该系统卡右上角的 `设置` 按钮后，进入全局设置页。

## 全局交互规则

### 启动与落点

- 首次进入 App，默认落在 `Tab01.信息页`。
- 非首次进入时，默认回到用户上次停留的 tab。
- 若用户是从训练中的系统通知或灵动岛入口进入，则直接进入训练中工作台，而不是首页壳层。

### tab 切换

- 点击底部 tab，切换到对应主页面。
- 切换 tab 时保留各自滚动位置和局部状态，避免高频来回跳转时丢失上下文。
- 手机布局下，支持在一级页面容器上横向滑动切换四个主 tab。
- 当手势起点位于图表、日历、横向列表或其他明确的横向交互控件内时，应优先由子控件消费手势，而不是强行切换 tab。
- 大屏 `navigation rails` 布局下，不再提供整页横滑切 tab。

### 训练进行中的优先逻辑

- 用户只要发起训练，首页壳层就进入“存在进行中训练”状态。
- 此状态下所有 tab 都显示统一的全局训练高亮条。
- 高亮条在 `训练中工作台` 内也继续显示。
- 高亮条的详细状态、计时与点击逻辑统一见 [首页.全局训练高亮条.md](/Users/heruoxin/dev/PRD/电子教练/03.%20设计概览/首页.全局训练高亮条.md)。
- 若训练暂停超过规则阈值，相关 AI 提示显示在训练中工作台内，不在首页壳层额外打扰。

## 全局视觉与组件约束

### 顶部导航区

- 标题采用较大字号，体现当前 tab 的页面身份。
- 副标题用于展示日期、周期阶段或状态摘要。
- 背景可轻微透明，但标题文字必须保持高对比度。
- 不在首页壳层固定放置独立的 `设置` 入口。

### 底部 tab bar

- 采用浮动式玻璃条，而不是贴底纯色栏。
- 图标和文字都需要有激活态。
- 激活态不只依赖颜色，还要通过填充、发光或轻微浮起表达当前页。
- 该组件仅用于手机等窄屏设备。
- 大屏设备改为左侧 `navigation rails`，不再保留同层底部 tab bar。

### 页面滚动容器

- 每个 tab 自己拥有滚动区。
- 首页壳层本身不滚动，避免顶部返回训练条和底部 tab bar 一起位移。
- 内容区需预留底部安全区，避免最后一张卡片被 tab bar 遮挡。

### 大屏展开方式

- 大屏下的二级页、详情页和对话展开，尽量优先采用“分栏展开”或“悬浮窗口”。
- 避免在 `Pad` 上仍沿用手机式整页推进，导致上下文频繁丢失。
- 只有当目标页面本身需要长流程沉浸式操作时，才允许覆盖式全屏进入。
