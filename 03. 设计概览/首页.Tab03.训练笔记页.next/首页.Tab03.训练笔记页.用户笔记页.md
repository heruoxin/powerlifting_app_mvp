# 首页.Tab03.训练笔记页.用户笔记页

## 页面定位

用户笔记页用于承载单条用户笔记的创建、阅读、编辑和保存。

它不是单独拆开的“查看页 + 编辑页”，而是一个统一的笔记工作页：

- 打开已有笔记时，默认进入可阅读的完整页面。
- 若用户开始输入，就地进入编辑状态。
- 有修改时再显式保存，不需要先点一个 `编辑` 按钮。

## 进入方式

- `训练笔记页 > 用户笔记` 中点击任意笔记卡片。
- `训练笔记页 > 用户笔记` 中点击 `新建`。
- `训练笔记页 > 用户笔记` 中点击 `+ 新建训练笔记`。
- 从 `计划页` 或训练中工作台长按训练项目后，选择 `写笔记`。

## 页面目标

- 用一页完成单条用户笔记的新建、阅读、编辑和保存。
- 让被引用的训练对象一眼可见，并且对象本身可直接跳转。
- 让笔记在真实训练场景中被快速写下，而不是被复杂流程打断。
- 保持与 `问AI` 的顺滑衔接，但不把笔记页做成聊天页的附庸。

## 设计原则

### 1. 阅读与编辑合一

- 打开笔记后，不再额外区分“阅读模式”和“编辑模式”两个页面。
- 页面默认以阅读友好的排版呈现，但内容区域可直接聚焦输入。
- `保存` 只在有未保存修改时变为可操作态。

### 2. 引用对象本身就是入口

- 笔记顶部展示的训练计划、训练项目、训练记录等引用 chip 本身可直接点击。
- 不再额外放置 `[查看引用对象]` 之类的中间按钮。
- 这样能让“写下这条笔记时我在看什么”保持更短的回跳路径。

### 3. 创建态尽量少打断

- 从训练计划或训练中长按项目进入 `写笔记` 时，应直接打开本页。
- 被引用对象自动带入，不要求用户重新选择。
- 若正处于训练中，不应因为创建笔记打断当前训练的返回路径。

### 4. 笔记始终可以 `问AI`

- 用户在本页选中文本片段时，应出现浮动 `问AI`。
- `问AI` 带走的是结构化上下文，而不是把全文粗暴塞进输入框。

## 页面结构

用户笔记页从上到下由五层组成：

1. 顶部导航区
2. 引用对象区
3. 标题区
4. 正文区
5. 底部状态区

## 页面说明

### 顶部导航区

- 标题按场景变化：
  - 新建态：`新建用户笔记`
  - 已有笔记：`用户笔记`
- 左上角 `返回` 返回 `训练笔记页` 的 `用户笔记` tab。
- 右上角 `保存` 只在内容或关联变化时高亮。

### 引用对象区

- 展示当前笔记所关联的对象 chip。
- 首版优先支持三类可见引用：
  - `训练记录`
  - `动作块`
  - `单组`
- chip 本身可直接跳转到对应页面或对象定位。
- 一条笔记可同时存在多个引用 chip。

### 标题区

- 标题可选。
- 若为空，则在列表页回退为正文首行摘要。

### 正文区

- 正文采用大面积连续输入区。
- 页面打开时优先保证阅读体验，不强制自动弹出键盘；但从 `写笔记` 新建进入时，可以直接聚焦输入。
- 正文允许选择文本，以触发 `问AI` 等浮动操作。

### 底部状态区

- 展示最近保存时间。
- 若当前笔记是在训练中创建或编辑，可额外显示轻量提示，例如“已关联当前训练”。

## 关键交互规则

### 新建与保存

- 从 `新建` 进入时，创建一条空白笔记草稿。
- 从 `写笔记` 进入时，创建一条带引用对象的草稿。
- 只有在用户点击 `保存` 后，草稿才正式写入本地数据。

### 已有笔记的直接编辑

- 用户打开已有笔记后，直接在同页修改内容。
- 不再提供单独的 `编辑` 按钮。
- 若用户离开时存在未保存改动，应提示：
  - `保存并返回`
  - `放弃修改`
  - `继续编辑`

### 自动关联

- 若本页是在训练中的上下文里被打开，只要该条笔记发生过新建或编辑，就应自动与当前 `training_record.uid` 建立关联。
- 若进入来源带有具体动作或组，则对应 `exercise_block.uid` 或 `set.uid` 也应保留在引用列表中。
- 若同一条笔记后来又在其他训练上下文中被编辑，可继续追加新的训练记录或动作/组关联。

### `问AI`

- 用户选中文本后，可直接 `问AI`。
- 跳转 `Tab04.AI教练页` 时至少带入：
  - `note_uid`
  - `note_title`
  - `selected_text`
  - 关联对象摘要

## 与其他页面的关系

- 上一级页面：`首页.Tab03.训练笔记页`
- 跨页跳转：
  - 点击引用对象 chip -> 对应 `计划页` 或训练中工作台对象
  - 选中文本后 `问AI` -> `首页.Tab04.AI教练页`

## 当前结论

用户笔记页首版至少应稳定支持以下能力：

1. 单页承载新建、阅读、编辑和保存。
2. 引用对象 chip 直接跳转。
3. 训练计划 `uid`、训练项目 `uid`、当前训练记录上下文的统一承载。
4. 训练中的自动关联补写。
5. 文本选择后的 `问AI`。
