# 首页.Tab03.训练笔记页

## 页面定位

训练笔记页用于承载训练相关的非结构化内容。

它不是泛备忘录，也不是单纯的 Markdown 文件浏览器，而是把“用户自己写下来的东西”和“AI 教练持续整理出的东西”稳定放在同一页里，并明确区分来源。

本页固定采用顶部 `2 tab` 结构：

1. `用户笔记`
2. `教练笔记`

补充说明：

- `用户笔记` 用于承载用户自己输入的训练备注、复盘、疼痛记录、动作感受等内容。
- `教练笔记` 用于承载 AI 自动整理或维护的 Markdown 记忆文件，例如 `日记.md`、`教练观察.md`。
- 之前放在 `AI教练页` 中的教练记忆查看能力，统一收敛到本页的 `教练笔记` tab，不再在 `Tab04` 内单独存在。

## 页面目标

- 让用户自己的原始表达与 AI 的长期观察保持清晰边界。
- 给训练总结、长期观察、日记等 AI 自动产出内容提供稳定落点。
- 让任何值得追问的笔记文本都能就地触发 `问AI`，而不是要求用户手动复述上下文。
- 让 `教练笔记` 既透明可看，又不与普通聊天线程混在一起。
- 让用户笔记能稳定关联训练记录对象，并统一使用稳定 `uid` 引用。

## 设计原则

### 1. 双顶部 tab 长期稳定

- 本页内部固定使用 `用户笔记 / 教练笔记` 两个顶部 tab。
- 不再把两类内容混排在同一信息流里。
- 顶部 tab 的作用是明确“内容来源”，不是做复杂信息架构。

### 2. 用户笔记优先服务训练复盘

- 用户笔记的核心价值是表达当时的主观感受、异常情况和动作理解。
- 它不应被设计成泛化待办或生活日记区。
- 笔记应支持关联训练记录与训练中的具体动作/组，但不要求每条都必须绑定训练对象。
- 一条笔记可以同时关联多条训练记录，以及这些训练记录下的多个动作或组。

### 3. 教练笔记承担记忆透明化职责

- `教练笔记` 是用户查看 AI 长期记忆文件的正式入口。
- 它应该像“教练工作笔记区”，而不是 debug 面板。
- 默认优先展示对用户最有解释价值的文件，而不是一次把所有文件堆出来。

### 4. 任意可选中的笔记文本都应能 `问AI`

- 用户在笔记页里，不应被迫重新输入一遍自己刚刚看到的内容。
- 长按或选中文本后，应出现浮动操作，其中必须包含 `问AI`。
- `问AI` 应把被选中的文本片段作为结构化上下文带入 `Tab04.AI教练页`。

### 5. 训练闭环优先于纯文档管理

- 本页中的笔记最终都应能回到 `计划`、`训练`、`信息摘要`、`AI讨论` 的主链路中。
- 因此，笔记卡片与 Markdown 文件都要保留到相关训练对象和 AI 话题的深跳能力。

### 6. 关联关系由笔记侧维护，并基于稳定 `uid`

- 用户笔记对训练对象的关联，应统一建立在数据结构中的稳定 `uid` 上。
- UI 中展示给用户的是可读标签，例如训练日、动作名、组次，不直接暴露生硬的 `uid`。
- 关联关系由笔记侧维护即可，不要求训练计划或训练记录对象反向持有一份笔记列表。
- 这样可以先把 `03.训练笔记` 的能力写清，再把更复杂的 `02.训练计划` 交互留到下一阶段展开。

## 页面结构

训练笔记页从上到下由四层组成：

1. 顶部标题区
2. 顶部 `2 tab`
3. 当前 tab 的主内容区
4. 底部主操作区

## 页面区块说明

### 顶部标题区

- 页面标题固定为 `笔记`。
- 右上角操作根据当前 tab 不同而变化：
  - `用户笔记` 中优先显示 `新建`
  - `教练笔记` 中优先显示 `筛选`

### 顶部 `2 tab`

- 顶部 tab 固定为：
  - `用户笔记`
  - `教练笔记`
- 切换时保留各自滚动位置。
- 不支持用户自定义 tab 顺序。

### `用户笔记` tab

- 以时间倒序展示用户创建或编辑过的笔记。
- 每条笔记至少展示：
  - 标题或首行摘要
  - 最近编辑时间
  - 正文预览
  - 关联对象摘要
- 用户笔记支持增删改查。
- 点击任意笔记卡片，进入 `用户笔记页`。
- 卡片中的关联对象 chip 本身可直接点击跳转，不需要额外的“查看引用对象”按钮。

### `用户笔记` 的关联模型

- 每条用户笔记都应有自己的稳定 `note_uid`。
- 训练相关对象的结构化关联当前统一采用：
  - `training_record.uid`
  - `exercise_block.uid`（可选）
  - `set.uid`（可选）
- 一条笔记允许同时关联多条训练记录和多个动作/组引用，重复关联应自动去重。
- UI 只显示可读标签，例如 `Base W3 Day 2`、`卧推 Top set`、`03/24 训练`，底层实际存储仍用对应 `uid`。

### 从训练计划或训练中写笔记

- 用户在 `计划页` 或训练中工作台，长按任意训练项目时，弹出选项至少包含：
  - `问AI`
  - `写笔记`
- 点击 `写笔记` 后，直接新建一条用户笔记。
- 新笔记进入编辑态时，顶部自动带入当前被引用对象的引用 chip。
- 若当前长按对象属于某次训练中的某个动作或某一组，则新笔记至少自动关联：
  - 当前 `training_record.uid`
  - 对应 `exercise_block.uid` 或 `set.uid`
- 计划态下未开始训练的笔记结构化关联规则，留待训练笔记模型阶段单独细化；当前先保留可读来源标签，不在这里提前定稿。
- 新建后的笔记在独立页面中直接可输入和保存，不再先经过一个只读中间态。

### `用户笔记` 的 `问AI` 规则

- 长按任意笔记正文，或在详情页选中任意文本片段，应出现浮动操作栏。
- 浮动操作栏首个推荐动作为 `问AI`。
- 点击 `问AI` 后，跳转 `Tab04.AI教练页`，并带入结构化上下文：
  - `source_page = notes_user`
  - `context_type = note_snippet`
  - `note_id`
  - `note_title`
  - `selected_text`
  - 必要的前后文摘要
- 进入 AI 教练页后，所选文本不直接塞进输入框正文，而是显示为可见的引用卡片或引用块。

### 训练中的自动关联规则

- 只要用户处于某一次训练中的状态，任何在该状态下新建或编辑过的用户笔记，都应自动与该次训练所对应的训练计划建立关联。
- 若该笔记原本没有关联当前 `training_record.uid`，系统应自动补上；若已存在，则不重复写入。
- 若用户是从某个具体训练项目进入笔记编辑，还应继续保留该项目级别的关联。
- 这意味着一条笔记可以随着多次训练过程被不断补充，并逐步关联到多条训练记录。

### `教练笔记` tab

- `教练笔记` 是 AI Markdown 记忆文件的可见入口。
- 顶部固定展示解释性 banner，说明这是什么、由谁维护、常见更新时间是什么。
- 默认展示：
  - `教练观察.md`
  - `日记.md`
- 支持用户手动展开查看：
  - `训练计划.md`
  - `用户特质.md`
  - `soul.md`
- `soul.md` 仅支持查看，不支持编辑。
- 除 `soul.md` 外，其他已存在的教练记忆文件可编辑，但首版不支持新增新文件。
- 点击任意文件卡片或段落入口，进入 `AI教练笔记页`。
- 文件列表态不单独提供 `编辑` 按钮；打开文件页后即可直接阅读，必要时也可直接编辑和保存。

### `教练笔记` 顶部 banner

- banner 固定在 `教练笔记` tab 顶部，优先于文件列表展示。
- banner 至少要解释三件事：
  - `AI 教练笔记` 是什么
  - 这些内容不等于用户原始笔记
  - AI 通常会在什么情况下更新笔记
- 首版建议写明的常见更新时间至少包括：
  - 训练完成后
  - 训练中事件后
  - 周期阶段结束后
  - 重要教练对话完成后
- banner 可提供 `了解更多` 或 `更新规则` 入口，但不需要在本页继续拆出复杂设置页。

### `教练笔记` 的继续追问规则

- 在 `AI教练笔记页` 中，用户可通过选中文本或页面操作发起 `问AI`。
- 发起 `问AI` 时，跳转 `Tab04.AI教练页`，并带入对应文件或段落作为结构化上下文。
- 若用户在 `教练笔记` 内选中文本片段，也同样应出现浮动 `问AI` 入口。

## 关键交互规则

### 新建用户笔记

- 点击 `新建`，进入空白笔记编辑态。
- 首版优先支持纯文本笔记，不要求富文本格式能力。
- 用户可选择是否关联当前训练或某次历史训练。
- 若该次新建是从训练计划或训练项目长按进入，则不再要求用户手动选择关联对象，系统应默认预填。
- 用户打开单条笔记后，进入独立页面查看，并可在同页直接编辑和保存，不再额外区分“查看页”和“编辑页”。

### 教练笔记的默认打开顺序

- 进入 `教练笔记` tab 时，默认先看到 `教练观察.md` 和 `日记.md`。
- 其他文件以折叠入口显示，避免首屏信息过载。
- 若从其他页面带着明确文件目标跳入，则优先定位到目标文件或目标段落。
- 用户打开单个教练笔记文件后，进入独立页面查看；对可编辑文件可在同页直接修改并保存，不再额外提供单独的 `编辑` 入口按钮。

### 文本引用的一致性

- 无论是 `用户笔记` 还是 `教练笔记`，被引用进入 AI 教练页时都应保留来源信息。
- AI 页中的引用卡片至少要显示：
  - 来源页
  - 文件名或笔记标题
  - 摘录文本
  - 深跳返回入口

## Tab03 内部下钻页关系

以下页面属于 `Tab03.训练笔记页` 的内部展开页，应写入 `首页.Tab03.训练笔记页.next`：

- 点击 `用户笔记` 中任意笔记卡片 -> `用户笔记页`
- 点击 `新建` 或 `+ 新建训练笔记` -> `用户笔记页`
- 从 `计划页 / 训练中工作台` 长按训练项目后选择 `写笔记` -> `用户笔记页`
- 点击 `教练笔记` 中任意文件卡片或段落入口 -> `AI教练笔记页`

子文档职责约定：

- `用户笔记页` 负责承载单条用户笔记的新建、阅读、编辑和保存。
- `AI教练笔记页` 负责承载单个 AI 教练笔记文件的阅读、直接编辑和保存；其中 `soul.md` 保持只读。

### 当前范围控制

- 本轮文档先只确认 `训练笔记` 侧的关联原则和入口，不提前展开 `训练计划页` 的完整结构设计。
- 因此这里先统一使用“`training_record.uid` + `exercise_block.uid(optional)` + `set.uid(optional)`”表达训练对象关联。
- 更细的训练对象层级、命名和编辑 UI，放到下一步 `02.训练计划` 文档中细化。

## 与其他页面的关系

- `信息页` 中的 `查看完整笔记`，优先跳转到本页 `教练笔记` tab 的对应摘要来源。
- `信息页` 中的 `快速备注`，优先跳转到本页 `用户笔记` tab。
- `计划页` 中长按任意训练项目，可直接发起 `问AI` 或 `写笔记`。
- 训练中工作台中长按任意训练项目，也可直接发起 `问AI` 或 `写笔记`。
- `计划页` 与训练中工作台都可以把训练计划 `uid` 与训练项目 `uid` 传给本页对应的用户笔记。
- `AI教练页` 负责承接本页发起的 `问AI`，但不再承担教练记忆文件的展示职责。

## 当前结论

训练笔记页首版至少应稳定支持以下能力：

1. `用户笔记 / 教练笔记` 双顶部 tab。
2. 用户笔记对训练计划与训练项目的 `uid` 关联。
3. 从 `计划页 / 训练中` 长按项目后直接 `写笔记` 或 `问AI`。
4. 训练态下编辑笔记时的自动关联规则。
5. 一条笔记关联多条训练计划记录与多个训练项目。
6. 选中文本后的浮动 `问AI`。
7. `教练观察.md` 与 `日记.md` 的默认展示。
8. `教练笔记` 顶部固定解释 banner。
9. `用户笔记页` 与 `AI教练笔记页` 两个二级页面。
10. 其他教练记忆文件的手动展开查看。
11. 从笔记内容深跳到 `AI教练页` 的结构化上下文对话。
