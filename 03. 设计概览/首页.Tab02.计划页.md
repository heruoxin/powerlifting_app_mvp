# 首页.Tab02.计划页

## 页面定位

`计划页` 应被重新定义为一个稳定的、长期存在的“训练时间轴页面”。

它不只是“看接下来练什么”，也不只是“翻历史记录”，而是用户理解当前周期状态的主入口：

- 上半部分承载已经发生的训练
- 下半部分承载接下来的计划
- 中间存在清晰的“已训练 / 未训练”分界线
- 页面顶部还能直接看到当前处于哪个 `W`、哪个阶段，以及这个 `W` 的关键数据

用户每次进入这个页面，应该都能很自然地回答三件事：

1. 我最近一次练了什么？
2. 我接下来要练什么？
3. 我现在处于这个周期的什么位置？

## 页面目标

- 让 `计划页` 成为用户回看训练历史、查看接下来计划、理解当前周期位置的稳定主入口。
- 让“最近完成训练”“下一次训练”“当前处于哪个 W / 哪个阶段”在默认进入时都可感知。
- 让用户可以通过顶部训练日过滤器，直接回看某个具体训练日标签的时间推进。
- 让中周期特征不再藏在二级页里，而是在主页面顶部即可进入。
- 保持“新建计划”和“开始训练”两个高频动作都足够顺手。

## 核心原则

### 1. 主页面仍是稳定视图，不是功能拼盘

- `计划页` 首先是一个“看训练安排与训练位置”的页面。
- 新建计划、开始训练等高优先级动作通过 FAB 进入，不抢占主内容结构。
- 页面本体尽量不堆过多独立卡片。

### 2. 默认停在分界线上

- 首次进入 `计划页` 时，默认使用 `时间轴视图`。
- 初始落点不在列表顶部，也不在底部，而是停在：
  - 最近一次已完成训练
  - 与第一条未训练计划
  - 之间的分界线附近
- 这样能确保“刚练完什么”和“接下来练什么”同时进入视野。

### 3. 中周期信息必须在主页面显性可达

- `计划页` 不应只展示零散训练日，而应让用户随时知道当前处于哪个 `W`。
- 当前 `W`、所属阶段、当前周性质等信息要在页面顶部显式呈现。
- 与当前 `W` 相关的关键数据，应通过点击顶部 `W 指示器` 快速查看，而不是要求用户自己翻时间轴推断。

### 4. 训练日过滤器直接服务回看

- 用户想看某一类训练日时，本质上仍是在看时间轴中的特定训练日标签。
- 因此时间轴顶部直接提供 `训练日过滤器`：
  - 默认 `全部`
  - 用户可切到 `D1`、`D2A`、`D2B`、`OS` 等当前计划里真实存在的训练日标签
- `D2A`、`D2B` 这类标签直接按字符串视为独立训练日，不做自动归并。

### 5. FAB 改为纵向双 FAB

- 页面右下角固定为纵向排列的两个 FAB：
  - 上方较大的 `新建计划`
  - 下方较小的 `开始训练`
- `新建计划` 负责创建新计划或新训练日。
- `开始训练` 继续负责进入下一次训练或相关选择流程。

## 页面结构

`计划页` 从上到下由五层组成：

1. 顶部标题区
2. 视图切换区
3. 当前视图的顶部工具栏
4. 当前视图的主内容区
5. 右下角纵向双 FAB

## 顶部布局

### 顶部标题区

- 页面标题固定为 `计划`。
- 不再在主页面顶部堆叠长段摘要卡。
- 只有当前视图必须依赖的控制信息，才放在标题下方。

### 视图切换区

- 顶部稳定提供两个视图切换：
  - `时间轴`
  - `日历视图`
- 切换后保留各自滚动位置与局部状态。

### 时间轴视图顶部工具栏

时间轴视图顶部固定采用一行左右结构：

```text
[ (i) W3 ]                                [全部]
```

规则说明：

- 左侧是 `W 指示器`：
  - 实时显示“主内容区里最上方那张可见训练卡”属于第几个 `W`
  - 随用户滚动时间轴实时变化
  - 点击后弹出当前 `W` 信息模态框
- 右侧是 `训练日过滤器`：
  - 默认 `全部`
  - 用户可切到当前计划里真实存在的训练日标签，例如 `D1A`、`D2A`、`D2B`、`D4B`
  - 若当前页面数据中存在开放训练，则过滤器也可显示 `OS`

补充说明：

- `W 指示器` 以“主内容区中最靠上的可见训练卡”为准，而不是固定显示当前主计划的活跃 `W`。
- 若顶部正好停在分界线位置，则取分界线下方第一张可见训练卡所属的 `W`。
- `训练日过滤器` 只影响时间轴主体的显示，不改变当前计划的活跃 `W`。

## 视图一：时间轴视图

### 页面职责

- 按时间顺序把训练日逐个排开。
- 同时展示已完成训练和接下来训练计划。
- 成为用户最常用、最稳定的默认视图。
- 在同一页面里承载“当前 W / 当前阶段 / 训练日标签过滤”。

### 顶部 `W 指示器`

- 默认显示主内容区最上方那张可见训练卡所属的 `W`，例如 `W3`。
- 左侧 `(i)` 用于强调它是“可点击查看更多周信息”的入口。
- 点击后弹出 `W 信息模态框`。

`W 信息模态框` 至少应展示：

- 当前选中 `W`
- 与它对应的中周期阶段，例如 `Hypertrophy / Strength / Peaking`
- 中周期日期范围
- 当前周状态，例如 `进行中 / 未开始 / 已完成`
- 当前周性质，例如 `regular / deload`
- 关键统计：
  - 共几练
  - 已完成几练
  - PR 突破
  - 疲劳度
  - 三项主项的组数、重量范围、总容量

补充说明：

- `W 指示器` 需要随滚动实时刷新。
- 只要最上方可见训练卡进入新的 `W`，顶部显示就要同步变化。

### 顶部 `训练日过滤器`

- 默认选中 `全部`。
- 用户切到某个训练日标签后，时间轴只展示该 exact string 对应的训练日。
- 若计划结构中存在 `D2A / D2B` 这类标签：
  - 过滤器应直接展示 `D2A`
  - 过滤器也应直接展示 `D2B`
  - 两者视为两个独立训练日
- 若时间轴中同时存在开放训练，过滤器也可出现 `OS`。

### 视觉规则

- 每一个训练日对应一个稳定的时间轴节点。
- 节点状态建议如下：
  - `已完成`：实心节点
  - `未训练`：空心节点
  - `进行中`：高亮节点
- 每张卡片都应显式带出该训练日当前所属阶段，例如：
  - `Hypertrophy`
  - `Strength`
  - `Peaking`

### 内容规则

- 每个训练日卡片以“动作块摘要”的形式展示，不在主页面完整展开所有组次。
- 摘要至少保留：
  - `W?`
  - `D?`
  - 日期或相对时间
  - 当前所属阶段
  - 动作名称
  - 最关键的负荷行
  - `...` 表示还有更多内容

### 初始落点规则

- 进入页面时，自动滚动到“已训练 / 未训练”分界线附近。
- 分界线以上至少保留最近一次已完成训练。
- 分界线以下至少保留最近一次待训练计划。
- 若当前已启用训练日过滤器，初始落点改为：
  - 最近一次已完成的该标签训练日
  - 与下一次未训练的该标签训练日
  - 之间的位置

### 进行中训练的显示规则

- 若时间轴中存在一条 `training_record.state = in_progress` 的训练：
  - 节点使用更强的高亮态
  - 卡片标题区显著显示 `进行中`
  - 卡片摘要优先显示当前进度，而不是只显示计划摘要

首版建议至少显示：

- 当前训练日标签
- `进行中`
- 已完成动作或组的概览，例如：
  - `已完成 2 / 6 组`
  - 或 `已完成 1 / 3 动作`
- 必要时可显示本次训练已用时

这样做的目的是：

- 用户回到 `计划页` 时，一眼就能找到正在进行中的那条训练
- 避免它和普通“未训练计划”长得太像

## 视图二：日历视图

### 页面职责

- 用月历方式帮助用户建立训练节奏感。
- 更适合回顾“这个月练了多少次、异常和亮点落在哪些天”。
- 在同一页面中承接“月摘要信息、单次训练回看、待训练计划预览”三类内容。

### 布局规则

- 布局参考系统日历 app 的月历布局。
- 页面整体分为上下两部分：
  - 上半部分固定为日历区
  - 下半部分为可竖向滚动的信息区
- 每个日期格内只放极简信息，例如：
  - `W3 D2A`
  - `深蹲`
  - 特殊角标
- 日历区默认打开当前分界线所在的月份。
- 下半部分在没有点击具体训练日时，默认显示“当月摘要信息”。
- 当月摘要应复用 `W 指示器` 模态框的核心结构，至少展示：
  - 所属中周期 / 月份范围
  - 当前月状态
  - 本月完成训练数 / 待训练数
  - PR、失败组、异常等关键摘要
  - 三项主项的月度统计
- 当用户点击某一个训练日时，下半部分应插入该次训练的训练卡片。
- 该训练卡片直接复用时间轴模式的训练卡片结构与状态表达，不单独创造新的卡片语法。
- 在当前这个月，或未来月份，下半部分继续显示待训练的计划卡片列表。
- 待训练计划卡片应出现在月摘要或已点击训练卡片之后，帮助用户同时看到“这个月练了什么”和“接下来要练什么”。

### 特殊角标规则

首版至少支持以下角标：

- `PR`
  - 表示当天出现 PR 突破
- `10`
  - 表示当天出现 `RPE 10`
- `!`
  - 表示当天出现组失败

补充说明：

- 若一天内同时命中多个角标，需定义优先级与堆叠规则。
- 角标应尽量放在日期格角落，不压住主要信息。

### 视觉规则

- 不同 `D` 可使用不同颜色标记，帮助用户快速识别周期位置。
- 颜色首先服务于训练日标签的识别，而不是服务于动作类型。
- 当前日期需要显著高亮。
- 下半部分的月摘要区、训练卡片区、待训练计划区应保持同一滚动上下文，避免分裂成多个局部滚动容器。

## FAB 规则

### 纵向双 FAB

- 页面右下角固定为纵向排列的两个 FAB：
  - 上方：`新建计划`
  - 下方：`开始训练`
- 上方 FAB 尺寸更大，因为它承载更明确的新建入口。
- 下方 FAB 尺寸更小，但仍需保持高可点按性。

### `新建计划`

- 点击后进入全屏选择页，而不是直接生成空白训练日。
- 全屏页从上到下固定给出三个选项：
  - `手动新建`
  - `选择计划复制`
  - `让 AI 自动安排`

三个入口的规则如下：

- `手动新建`
  - 直接进入空白训练日编辑页
- `选择计划复制`
  - 先选择一个既有训练日，再复制出新的训练日草稿供微调
- `让 AI 自动安排`
  - 跳转 `AI教练页`
  - 预置问题可直接使用：
    - `帮我安排下周的训练计划`
  - AI 应优先结合：
    - `训练计划.md`
    - 现有训练记录
    - 当前周期位置
    - 近期执行情况

### `开始训练`

- 默认指向时间顺序上“下一次未训练”的训练日。
- 若只有一个明确的下一次训练，直接进入该训练的 `训练中工作台`。
- 若同时存在多个同优先级的未训练训练日，则先弹出轻量选择 sheet。
- 若用户这次不想进入既定计划，首版仍应保留进入 `随便练练 / OS` 的次级入口。

### 存在进行中训练时

- 若已经有进行中的训练，则下方小 FAB 优先替换为 `继续训练`。
- 不允许在已有进行中训练时再无提示开启另一条新训练。

## 点击规则

### 点击未训练计划

- 进入 `计划详情与编辑页`。

### 点击已结束训练

- 进入 `训练完成页`。

### 点击进行中训练

- 进入 `训练中工作台`。

## 页面状态规则

### 无历史、无未来计划

- 主内容区显示空态。
- 仍保留双 FAB：
  - `新建计划`
  - `开始训练`
- 当没有任何未训练计划时，点击 `开始训练` 应提示：
  - 当前没有可开始的计划
  - 可改用 `随便练练`

### 只有历史，没有未来计划

- 分界线下方显示空态提示。
- 提示用户当前没有待训练计划。
- `新建计划` 此时应成为更显性的推荐入口。

### 只有未来计划，没有历史

- 分界线以上为空态。
- 用户仍应一进页面就看到“最近要练什么”。

## Tab02 内部下钻页关系

以下页面仍属于 `Tab02.计划页` 的内部展开页，应写入 `首页.Tab02.计划页.next`：

- 点击未训练训练日 -> `计划详情与编辑页`
- 点击进行中的训练日 -> `训练中工作台`
- 点击已结束训练日 -> `训练完成页`

补充说明：

- 这三个页面不应被理解为三套彼此割裂的页面。
- 更准确地说，它们是同一套复杂页面在三种训练状态下的展开：
  - 未开始 / 可编辑
  - 训练进行中
  - 训练已结束 / 可复盘

## 当前结论

`计划页` 这一版主页面首要确认以下几点：

1. 它仍是一个稳定的、类似时间轴与月历结合的主视图页面。
2. 默认视图是 `时间轴`，默认落点在已训练与未训练分界线附近。
3. 页面顶部通过训练日过滤器直接支持按标签回看时间推进。
4. 页面顶部必须显式提供 `W 指示器`，并能弹出包含阶段与关键数据的周信息模态框。
5. 日历视图中要增加 PR、RPE10、组失败等特殊角标。
6. FAB 改为纵向双 FAB：上方 `新建计划`，下方 `开始训练`。
7. `新建计划` 的首个全屏入口固定为：手动新建、选择计划复制、让 AI 自动安排。
8. `D2A / D2B` 这类标签直接按字符串视为独立训练日，不做自动归并。
9. `W 指示器` 以屏幕里最上方的可见训练卡为准，并随滚动实时变化。
10. 页面需要同时服务“看最近一次训练”“看下一次训练”“看当前处于哪个周期位置”三类需求。
