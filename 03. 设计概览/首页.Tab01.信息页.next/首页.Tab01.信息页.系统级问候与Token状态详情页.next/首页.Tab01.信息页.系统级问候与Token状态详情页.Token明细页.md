# 首页.Tab01.信息页.系统级问候与Token状态详情页.Token明细页

## 页面定位

该页面用于展示 `AI Token` 的流水明细。

目标是让用户理解“消耗发生在哪里”，提高计费透明度，而不是只给出一个余额数字。

## 进入方式

- 从 `Token 状态详情页` 点击 `消耗明细` 进入。
- 从 `最近消耗来源` 卡片点击 `查看全部明细` 进入。

## 页面结构

1. 顶部导航区
2. 筛选与时间范围区
3. 明细列表
4. 结算解释区

## 页面说明

### 筛选与时间范围区

- 允许按类型筛选：
  - `全部`
  - `购买`
  - `对话`
  - `后台总结`
  - `计划讨论`
- 允许切换时间范围。

### 明细列表

- 每条记录至少包括：
  - 时间
  - 事件名称
  - 消耗或充值数值
  - 对应上下文对象的简要引用
- 消耗使用负值，充值使用正值。

### 结算解释区

- 页面底部提供简短说明：
  - 消耗与任务复杂度相关。
  - 预测可用时长只是估算。
  - 若用户发现异常，可从这里发起帮助或反馈。

## 交互规则

- 点击某条明细，优先跳去对应对象：
  - 对话记录 -> `Tab04.AI教练页`
  - 训练总结 -> `Tab03.训练笔记页`
  - 周期讨论 -> `Tab04.AI教练页`
- 充值类记录不继续下钻。
