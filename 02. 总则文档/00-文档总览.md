# AI文档总览

## 文档目的

`AI文档` 是本项目后续落地时使用的正式文档区，用于沉淀经人工与 AI 协作确认后的产品、需求、数据和交互设计内容。

本文档区的目标不是记录灵感，而是形成可持续迭代、可交付、可供后续开发直接参考的项目文档。

## 当前项目理解

基于现有 `index.md` 与 `人工文档`，当前项目可先理解为：

- 一款面向力量举专项训练者的训练记录 APP。
- 产品核心围绕训练计划执行、训练中记录、训练后复盘、AI 教练分析展开。
- 重点用户不是泛健身用户，而是已经接触过专项训练方法、对动作质量和训练负荷有一定理解的用户。
- 首发仅开发 `iOS`、`Android` 版。
- 实现上约束为统一 web 技术栈开发，并由移动端原生壳承载，以保留后续跨平台扩展空间。
- 用户训练数据与 AI 记忆采用本地优先存储，并支持未来进行整体迁移、导出和导入。
- 服务端应保持极简，主要承担 AI API 中转、点数计费账号体系，以及 AI 调用时的能力注入职责。
- 当前阶段仍处于文档设计期，尚未进入实际开发。

## 文档编写原则

- 以落地为导向：优先写清楚功能边界、用户流程、数据结构和 AI 行为。
- 以力量举场景为核心：避免被泛健身场景稀释。
- 以中高频使用场景为中心：优先覆盖训练前、训练中、训练后和长期复盘。
- 有结论，也保留待确认项：凡是尚未确认的设计，统一进入待确认清单，不在正式文档中假装定稿。

## 当前协作方式

- `人工文档`：保留原始思考，AI 只读不写。
- `AI文档`：沉淀共同确认后的正式文档。
- 每一轮协作优先做一件事：
  1. 确认一个主题的边界。
  2. 起草对应文档。
  3. 根据你的反馈修订为更稳定版本。
