# AI记忆文件模板与更新规则

## 文档目的

本文档用于把 AI 教练的记忆体系进一步细化到“每个文件大致写什么、什么时候更新、如何被加载”的层级。

本文档是模板级草案，目标是让后续实现与继续讨论时有统一基准。

## 记忆文件总览

当前已确认采用以下五份 Markdown 文件：

1. `soul.md`
2. `训练计划.md`
3. `用户特质.md`
4. `教练观察.md`
5. `日记.md`

## 文件一：`soul.md`

### 作用

- 定义教练是谁。
- 定义教练的目标任务、性格与风格。
- 定义教练的工作原则与安全边界。

### 更新原则

- 每次任务默认加载。
- 更新频率最低。
- 不因单次训练结果而频繁改动。
- 在产品运行态视为只读，不作为用户可编辑文件。

### 推荐模板

```md
# 教练使命

## 核心目标
- 帮助用户在力量举训练中更稳定地执行、复盘和调整计划

## 教练角色
- 你的定位是什么
- 你不是什么

## 工作原则
- 优先长期进步
- 优先可执行建议
- 优先结合用户真实记录判断

## 交流风格
- 专业但不过度堆术语
- 对不懂的概念主动解释

## 风险边界
- 不擅自修改已结束训练记录
- 计划修改需用户确认

## 工具使用原则
- 优先读取必要数据再判断
- 通过受控 skill 操作产品对象
```

## 文件二：`训练计划.md`

### 作用

- 描述当前周期所处阶段。
- 记录当前周期的设计思路与安排意图。
- 记录近期计划调整的理由。

### 更新原则

- 涉及计划讨论时优先加载。
- 用户确认计划变更后更新。
- 不记录逐次事件流水，避免和 `日记.md` 重复。

### 推荐模板

```md
# 当前训练周期

## 当前阶段
- 例如：第 3 周，第 2 训练日

## 周期目标
- 当前周期主要想解决什么

## 设计思路
- 强度安排思路
- 容量安排思路
- 主项与辅助项关系

## 当前重点
- 最近两周的训练重点

## 已确认调整
- 调整了什么
- 为什么调整

## 下个检查点
- 下次应该重点观察什么
```

## 文件三：`用户特质.md`

### 作用

- 存放相对稳定的用户画像。
- 沉淀身体状态、偏好、习惯、伤病与恢复特征。

### 更新原则

- 默认加载。
- 只写相对稳定或重复出现的特征。
- 不把单次事件直接写进这里。

### 推荐模板

```md
# 用户特质

## 身体状态
- 身高体重级别
- 某些长期存在的身体特征

## 动作特点
- 深蹲/卧推/硬拉的常见问题

## 偏好与习惯
- 喜欢的训练节奏
- 不喜欢的安排

## 伤病与风险点
- 已知伤病
- 需要长期注意的部位

## 恢复特征
- 恢复快慢
- 睡眠/疲劳的典型表现

## 沟通特点
- 更适合怎样的解释方式
```

## 文件四：`教练观察.md`

### 作用

- 存放近期到中期的趋势判断。
- 承接从 `日记.md` 提炼出的结论。
- 连接事件流与较稳定用户特质。

### 更新原则

- 长期趋势讨论时优先加载。
- 可按训练完成、周期结束、连续异常后更新。
- 应比 `日记.md` 更浓缩，但比 `用户特质.md` 更贴近当前阶段。

### 推荐模板

```md
# 教练观察

## 当前阶段判断
- 当前整体状态如何

## 近期趋势
- 主项表现趋势
- 疲劳趋势
- 执行质量趋势

## 已确认观察
- 有较强证据支持的结论

## 待验证假设
- 暂时怀疑但还需更多证据的判断

## 当前风险点
- 近期最需要注意的事

## 接下来关注点
- 下阶段要继续观察什么
```

## 文件五：`日记.md`

### 作用

- 作为事件型追加日志。
- 记录训练完成、重要对话完成等关键事件后的总结。

### 更新原则

- 以追加为主，不回写旧条目。
- 作为原始素材，供其他文件提炼。
- 不要求每次全量加载。

### 推荐模板

```md
# 日记

## 2026-03-20 训练完成

### 触发来源
- 训练完成

### 事件摘要
- 今天完成了什么

### 关键观察
- 表现如何
- 有什么异常

### 用户状态
- 主观疲劳
- 疼痛/不适

### 后续建议
- 下一次训练需要注意什么

### 是否需要提炼
- 用户特质：否
- 训练计划：是
- 教练观察：是
```

## 更新链路建议

### 训练完成后

- 追加一条 `日记.md`
- 生成本次训练总结
- 视情况更新 `教练观察.md`

### 完成一次重要 AI 教练沟通后

- 追加一条 `日记.md`
- 若对周期理解、用户特质或长期判断产生稳定变化，再更新中层文件

### 周期阶段结束后

- 追加一条阶段总结到 `日记.md`
- 优先更新 `训练计划.md` 与 `教练观察.md`

### 用户确认计划调整后

- 更新 `训练计划.md`
- 必要时在 `日记.md` 追加一条计划变更记录

### 形成稳定用户画像后

- 更新 `用户特质.md`

## 加载规则建议

### 默认加载

- `soul.md`
- `用户特质.md`

### 按需加载

- 计划相关任务：`训练计划.md`
- 长期趋势相关任务：`教练观察.md`
- 事件回溯：读取 `日记.md` 相关片段

## 与 `教练笔记` 的关系

当前已确认：

- 默认展示 `日记.md`
- 默认展示 `教练观察.md`
- `soul.md`、`训练计划.md`、`用户特质.md` 支持手动展开查看
- `soul.md` 仅支持查看，不支持编辑

补充说明：

- 上述展示入口位于 `训练笔记页 > 教练笔记`。
- `AI教练页` 不再直接承担这些记忆文件的查看职责。

## 当前待确认问题

- “重要 AI 教练沟通”的判定标准。
- `教练观察.md` 的更新频率是否需要单独设上限。
